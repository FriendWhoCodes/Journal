Man of Wisdom is evolving from a wisdom blog into a **philosophy-powered productivity ecosystem**. Everything we build helps people live more intentionally through ancient wisdom applied to modern tools.

## The Man of Wisdom Philosophy

### Brand Pillars

1. **Ancient Wisdom, Modern Application**
   - We don't just share quotes; we build tools to practice them
   - Stoicism, Eastern philosophy, timeless truths

2. **Intentional Living**
   - Every product asks: "How does this help you live with more purpose?"
   - Anti-distraction, pro-reflection

3. **Beautiful Simplicity**
   - Clean design, no bloat
   - Wisdom is simple; tools should be too

4. **Time as Sacred**
   - Life is finite; time is precious
   - Our tools help you use time wisely and remember it well

### Brand Voice

- Thoughtful, not preachy
- Warm, not corporate
- Wise, not condescending
- Encouraging, not pushy

**Pros:**
- Shared code (auth, UI, database)
- One place for everything
- Easier refactoring
- Consistent dependencies

**Cons:**
- Larger repo
- Need good organization

**Option B: Separate Repos (Your Current)**
```
manofwisdom-journal/    # Goals + Journal (current)
manofwisdom-time/       # Time Views (new)
manofwisdom-web/        # Main website (new)
```

**Pros:**
- Clear separation
- Independent deployments
- Smaller codebases

**Cons:**
- Duplicated code (auth, UI)
- Harder to share components
- More repos to manage

### Recommendation

**Short-term (Now):** Keep current structure
- `manofwisdom-journal` = Goals + Journal + Time Views
- Add Time Views to existing repo
- Share auth, database, components

**Long-term (When it grows):** Consider monorepo with Turborepo
- When you have 5+ apps
- When shared code becomes significant
